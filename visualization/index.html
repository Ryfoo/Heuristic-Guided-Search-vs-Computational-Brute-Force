Key Findings
01
BFS GUARANTEES OPTIMALITY
BFS always finds the shortest path in an unweighted graph. In 3D with branching factor 6, it expands the full frontier before going deeper.
02
GREEDY IS FASTEST, NOT OPTIMAL
Greedy best-first expands the fewest nodes by always following the heuristic, but sacrifices path quality — cost can be significantly higher.
03
A* DEGRADES IN DENSE SPACE
In highly connected 3D mazes, Manhattan distance provides weak discrimination. A* approaches Dijkstra behavior with priority queue overhead.
04
DFS FINDS LONG PATHS FAST
DFS explores deep corridors quickly but produces extremely suboptimal paths. Path length can be 100× longer than optimal in 3D mazes.
